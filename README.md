# nova Flashcards ✨

Un'applicazione per flashcards moderna, immersiva e dalle alte prestazioni, costruita con React e Vite. L'interfaccia segue i principi del **Sistema Visivo Universale** con un'estetica "Dark nova", combinando sfondi profondi, superfici in vetro (glassmorphism) e animazioni fluide per un'esperienza utente premium.

## 🚀 Caratteristiche Principali

- **Design "Dark Cosmic"**: Interfaccia utente elegante e immersiva, progettata con cura per la gerarchia visiva.
- **Glassmorphism**: Componenti e schede semitrasparenti che si integrano perfettamente con lo sfondo novao.
- **Animazioni Fluide**: Micro-interazioni naturali e transizioni morbide per un feedback visivo istantaneo.
- **Stack Moderno**: Sviluppata con React 19, React Router v7, e Vite per prestazioni eccezionali.
- **Iconografia**: Utilizzo di Lucide React per icone coerenti e scalabili.

## 🛠️ Tecnologie Utilizzate

- **Framework**: [React](https://react.dev/) + [Vite](https://vitejs.dev/)
- **Routing**: [React Router](https://reactrouter.com/)
- **Icone**: [Lucide React](https://lucide.dev/)
- **Gestione ID**: [UUID](https://github.com/uuidjs/uuid)
- **Stile**: CSS puro basato su design system personalizzato (vedi `DESIGN.md`).

## 📦 Installazione e Avvio Locale

1. Clona il repository:
   ```bash
   git clone https://github.com/tuo-username/nova-flashcards.git
   ```
2. Accedi alla directory del progetto:
   ```bash
   cd nova-flashcards
   ```
3. Installa le dipendenze:
   ```bash
   npm install
   ```
4. Avvia il server di sviluppo:
   ```bash
   npm run dev
   ```
5. Apri il browser all'indirizzo mostrato nel terminale (solitamente `http://localhost:5173`).

## 🎨 Design System

Il progetto segue rigorosamente le linee guida definite in `DESIGN.md`. 
La modalità scura (Dark Mode) è la modalità primaria, caratterizzata da sfondi novai (`--bg-deep`, `--bg-base`) e superfici semitrasparenti in vetro. Per maggiori dettagli su colori, tipografia, ombre e spaziature, consulta il file `DESIGN.md` incluso nel repository.

## 📜 Script Disponibili

- `npm run dev`: Avvia l'ambiente di sviluppo.
- `npm run build`: Crea la build ottimizzata per la produzione.
- `npm run lint`: Esegue il linter sul codice per garantire la qualità.
- `npm run preview`: Avvia un server locale per testare la build di produzione.
